# Apxu.FLASH CSS
Данный проект представляет собой не просто набор шаблонных часто используемых блоков, но и современную методику написания CSS.
Более подробно про архитектуру CSS, разработку этой методики, более сложные макеты и объяснения кода на канале Boosty.
Там же есть первый основной курс **"HTML5 and CSS"** для продолжающих по всем основным темам современного CSS.
Здесь же будет более кратко написано всё самое необходимое.
## Кратко о стиле написания
При написании стилей помимо архитектурной структуры проекта следует придерживаться следующих правил:
* Максимальная специфичность почти всех селекторов не должна превосходить специфичность класса 010. Это легко делается с помощью селектора обнуления специфичности **:where()**. В небольшом количестве случаев допустима специфичность 011 (например, **input[type="text"]** или **.elem::before**) и двойного класса 020 (например, класс с псевдоклассом **.block-name__elem:hover**).
* Использование CSS вложенности (**CSS Nesting**). Легко привыкнуть после небольшой практики, всё есть в примерах кода.
* Написание медиа и контейнерных запросов преимущественно с минимальной шириной **(min-width: ..)** или **(inline-size >= ..)** (в соответствии с CSS вложенностью внутри селектора). Для устройств с наведением используем **@media (hover)**.
* Использование переменных для связанных значений стилей с комментированием неочевидных мест. Подробнее в файлах шаблонных блоков **main.css**.
* Какие-либо другие правила, которые используются у меня в шаблонных блоках и при реализации простого полноценного макета в папке **easy-layout**.
## Структура проекта
Шаблон для нового проекта или для переработки уже имеющегося находится в папке **acss**. Он включает 6 основных CSS файлов, часть которых подключается по необходимости.
### normalize.css
Данный файл написан не мной, он представляет собой предварительную нормализацию CSS для одинакового отображения во всех браузерах, при этой сохраняет браузерные стили (user agent stylesheet). Подключается первым.
### fonts.css
Этот файл содержит все подключаемые локально шрифты. Все шрифты рекомендуется использовать в формате **.woff2**. Для Google шрифтов я брал имеющийся удалённый файл и копировал оттуда правило **@font-face**. При этом для удобства все шрифты обрамляются в комментарии. Пример:
```css
/**
 * Open Sans, sans-serif
 */

/* cyrillic */
@font-face {
  font-display: swap;
  font-family: 'Open Sans';
  font-stretch: 100%;
  font-style: normal;
  font-weight: 300 800;
  src: url('fonts/open_sans/cyrillic.woff2') format('woff2');
  unicode-range: U+0301, U+0400-045F, U+0490-0491, U+04B0-04B1, U+2116;
}

/* latin */
@font-face {
  font-display: swap;
  font-family: 'Open Sans';
  font-stretch: 100%;
  font-style: normal;
  font-weight: 300 800;
  src: url('fonts/open_sans/latin.woff2') format('woff2');
  unicode-range: 
    U+0000-00FF, U+0131, U+0152-0153, U+02BB-02BC, U+02C6, U+02DA, U+02DC, U+0304, U+0308, 
    U+0329, U+2000-206F, U+20AC, U+2122, U+2191, U+2193, U+2212, U+2215, U+FEFF, U+FFFD;
}
```
Для общего шрифта разумно оставить комментарий **/* all */**.
### base.css
Содержит все основные базовые стили и переменные на весь сайт, этот файл следует настраивать под каждый проект. Подробно про все стили можно прочитать в комментариях к файлу. Вот пример определения глобальных переменных на проект, которые разделены на группы:
```css
:root {
  /* Colors */
  --color-main-1: #212121;
  --color-main-2: #dedede;

  /* Fonts */
  --font-main-sans-serif: 'Open Sans', 
    system-ui, -apple-system, blinkmacsystemfont, 
    'Segoe UI', roboto, helvetica, arial, sans-serif;
  --font-main-serif: 'Roboto Slab', 
    georgia, 'Times New Roman', 
    'Liberation Serif', times, serif;
  --base-fs: 16px;
  --base-fs-number: 16; /* Дублирует число размера шрифта без 'px' */
  --base-lh: 1.5;

  /* Offsets */
  --offset-base: 20px;
  --offset-section: clamp(60px, 90 * 100vw / 768, 120px);
  --offset-form: 2em;

  /* Width */
  --width-base: 1160px;

  /* Time */
  --tr-dur-base: 0.3s;

  /* z-index */
  --z-index-default: 1;
}
```
Также в данном файле убираются все ненужные отступы, стилизуются заголовки всех требуемых уровней, простейшие стили для изображений, видео, диалогового окна, также содержит продвинутые стили для базовой стилизации форм. По необходимости может дополняться стилями по-умолчанию на весь сайт.
### main.css
Основной файл стилей. На каждый основной переносимый блок должен быть собственный класс. Стили пишутся в стиле BEM, но с нюансами под новую систему вёрстки.
* CSS класс для имени блока. Например, .block-name. Не должен содержать __, _ или --. HTML элемент с классом CSS блока можно переносить в любое место на странице.
* Вложенный элемент. Существует только внутри блока, отделяется символами __. Например, .block-name__elem. Существует внутри блока.
* Модификатор. Задаётся классом на блоке, отделяется символом _, например, .block-name_mod. Достаточно одного модификатора блока чтобы поменять весь блок.
В файлах **main.css** полно примеров блоков, все они реализуются в соответствии с основными правилами написания выше.
Каждый блок в файле отделяется маркерами комментариями. Таким образом легко их перетаскивать в любое нужное место.
```css
/**
 * .header BEGIN
 */

/**
 * .header END
 */
```
Если по-разному стилизуются теги внутри блоков, то родительский блок размещается выше дочернего, нет надобности увеличивать специфичность.
Совсем необязательно каждому тегу задавать класс элемента. Можно стилизовать по-умолчанию любые теги внутри блока. Например:
```css
.form-1 {


  :where(&) :is(input[type="text"], input[type="email"]) {

  }

  :where(&) label {

    &:first-of-type,
    &:first-child {

    }
  }

  :where(&) textarea {

  }
}
```
Классы с определёнными префиксами подразумевают соответствующую структуру. Например, у меня классы с префиксами **.list-** и **.menu-** подразумевают что это список **&lt;ul&gt;** или **&lt;ol&gt;**, который обязательно содержит дочерние элементы **&lt;li&gt;**.
Таким образом можно обойтись без классов элементов, например:
```css
.list-numbers-1_overlay {
  :where(&) > li {


    &::before {

    }
  }
}
```
При внесении какого-то шаблонного блока на какой-то конкретный сайт достаточно скопировать нужный блок и внести в него соответствующие изменения. Весь блок располагается в одном месте, поэтому это будет легко сделать.
### groups.css
Данный файл содержит "говорящие" стили по группам и пока что два блока общего назначения.
Каждые группы стилей отделяются маркерами и пишутся внутри. Например:
```css
/**
 * Section Block Model BEGIN
 */

.mb-0 {
  margin-bottom: 0;
}

.mt-24px {
  margin-top: 24px;
}

.mb-2em {
  margin-bottom: 2em;
}

.mb-1ob {
  margin-bottom: var(--offset-base);
}

.p-10px {
  padding: 10px;
}

.pb-1em {
  padding-bottom: 1em;
}

/* 'ob' выдуманные единицы измерения. 1ob -- 1 единица базового смещения в переменной '--offset-base' на :root */
.p-1ob {
  padding: var(--offset-base);
}

/* 'os' выдуманные единицы измерения. 1os -- 1 единица смещения в переменной '--offset-section' на :root */
.py-1os {
  padding-block: var(--offset-section);
}

/* 2os / 3 */
.py-2os-to-3 {
  padding-block: calc(2 / 3 * var(--offset-section));
}

.mt-20px {
  margin-top: 20px;
}

/* Безразмерная величина означает любой отступ, это просто номер */
.py-1 {
  padding-block: 38px;

  /* Контейнер с именем 'banners' уже есть в файле 'main.css', по необходимости можно сменить имя */
  @container banners (inline-size >= 944px) {
    padding-block: 76px;
  }
}

/**
 * Section Block Model END
 */
```
Названия классов интуитивно понятны, они в качестве дополнения, когда делать модификатор избыточно.
**Блоки общего назначения**
Вначале этого файла есть блок общего назначения **.layout-N** с элементами **.layout-N__cols-n**.
**N** это ширина одной колонки, **n** количество колонок. Напримеп, с помощью модификатора **.layout-250__cols-3** элемент внутри **.layout-250** делится не более чем на 3 колонки с минимальной шириной в 250px. Благодаря контейнерным запросам блок корректно отображается на любом viewport и внутри любого блока.
Ещё есть блок общего назначения с классом **.container**. Вот его код:
```css
/**
 * .container BEGIN
 */

.cq-1--container {
  container: container / inline-size;
}

.container {
  padding-inline: var(--offset-base);

  /* (inline-size >= var(--width-base) + 2 * var(--offset-base)) */
  @container container (inline-size >= 1178px) {
    padding-inline: calc(50% - var(--width-base) / 2);
  }
}

/**
 * .container END
 */
```
Если нет надобности в контейнерных запросах, то можно реализовать аналогичный дополнительный блок **.container-media** через медиа запросы.
В нём должна использоваться переменная базовой ширины и базового отступа.
### animation.css и другие
Как следует из названия в файле **animation.css** размещаются правила CSS анимации **@keyframes** с краткими комментариями. По необходимости можно создать другие .css файлы с общим смыслом или разделить файл **main.css** на несколько при большом количестве блоков или компонентной реализации сайта.
